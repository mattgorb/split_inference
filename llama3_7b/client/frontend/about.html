U-Shaped Split-Inference

A user prompt reaches the client API where it undergoes tokenization and computation through a few attention blocks. The client server runs on limited GPU resources and is meant to serve a small portion of the LLM. The output embeddings of the client model are obfuscated with noise Œ∑ and sent to the model owner, which runs on a separate server cluster. After processing the embeddings through the remaining attention blocks, the model server communicates the embeddings back to the client server to perform next token prediction. The implementation uses torch.distributed with Nvidia NCCL backend to perform node communication across remote GPUs.
